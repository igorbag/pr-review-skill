# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm test                          # run all tests (Node built-in runner)
node --test test/cli.test.js      # run single test file
node bin/cli.js init --yes        # run CLI locally
```

No build step. ESM throughout (`"type": "module"`), Node ≥ 18.

## Architecture

Two distinct layers, both in this repo:

**Layer 1 — Installer CLI** (`bin/`, `src/`): npm package that installs the skill into user repos.

**Layer 2 — Skill content** (`skill/`): the canonical PR review skill files bundled inside the package. These are what gets copied into user repos at `<canonicalDir>/`.

### Installer flow

```
bin/cli.js          → parses args, dispatches to src/commands/
src/commands/init   → detectTools + copySkill + writeAdapter
src/commands/update → copySkill(force) + preserve PROJECT_PROFILE.md
src/commands/doctor → read-only presence checks
src/lib/detect.js   → detects .claude/.cursor/.github dirs in user's cwd
src/lib/paths.js    → DEFAULT_CANONICAL_DIR, ADAPTER_RELPATHS, resolver fns
src/lib/skillfs.js  → SKILL_SRC_DIR = skill/, listSkillFiles(), copySkill()
```

`SKILL_SRC_DIR` is resolved relative to `skillfs.js` at `../../skill` — this is how the bundled skill content is located.

### Adapter pattern

Adapters (`.cursor/rules/pr-review.mdc`, `.github/instructions/pr-review.instructions.md`) are tiny pointer files — they just tell the AI tool "read `<canonicalDir>/SKILL.md`". They are generated by `init`, never touched by `update`.

### PROJECT_PROFILE.md invariant

`PROJECT_PROFILE.md` is never bundled in `skill/` and is never written by `copySkill`. It's generated at runtime by the AI during the first review. `update` preserves it byte-for-byte. This invariant is enforced in both `skillfs.js` (guards the copy loop) and `update.js` (read-before/restore-after).

### Tool detection

`detectTools(cwd)` checks for `.claude`, `.cursor`, `.github` directory existence. Adapters are only generated for tools whose root dir is present.

### `.claude/skills/pr-review/` in this repo

This is an installed copy of the skill used for self-hosted reviews of this repo's own PRs. It mirrors `skill/` but is separate — `skill/` is the source of truth that gets packaged and shipped.

## Test structure

`test/cli.test.js` uses Node's built-in `node:test` runner. Tests spin up real temp dirs (`os.tmpdir()`), invoke the CLI via `execFileSync`, and assert on exit codes + stdout/stderr. Requirements are tracked as R1–R12 in test descriptions.
